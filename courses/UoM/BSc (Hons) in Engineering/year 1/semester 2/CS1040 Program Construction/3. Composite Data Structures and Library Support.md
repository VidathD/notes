
Queues have 2 addresses. One to enter. One to exit.

Sets have no addresses.

Number of addresses on lists depends on the number of items.

Hash tables don't allow checking whether two items are the same. 
Therefore, we can't simply use a hash table to implement a set. We need to store multiple items in a single location pointed to by a hash.
When getting items, we can take a random string, hash it and if there is an item there, return it. If not, we can sequentially go down the addresses till we find one. Will need to keep a count of elements to prevent searching for an item in a null set.

If we have data that has natural elements such as geographical locations, a graph is probably the best way to represent it. Hash tables provide very fast access to items.

How to implement a map using hash tables? We can also implement a map using a graph.

We don't have a list set because it is very inefficient. Need to check the entire list one by one to see whether an item is there.
